# collect-html

把生成的 HTML 产物收进一个本地可浏览的面板。真文件留在它的项目里；collect-html
只在 `~/collect-html/` 下保存一个 **symlink + 一条 manifest 记录**。移除一条记录
只断开 symlink —— 项目里的真文件永不被动。

脚本承担所有确定性机械活，调用前缀（`$ROOT` 为 `${CLAUDE_PLUGIN_ROOT}/skills/collect-html/scripts`）：

```bash
bun "$ROOT/collect-html.ts" <command> ...
```

## Quick start —— 收藏刚生成的 HTML

1. 找到产物：刚写出的那个 `.html`（绝对路径）。
2. 自己推导元数据（这一步需要上下文，脚本做不了）：
   - **title** —— 取文件 `<title>`，没有就起一个简短人类可读标题。
   - **project** —— 该文件所在 repo/cwd 的目录名。
3. 判断 **keep vs throwaway** 并给出建议（默认 throwaway）：
   - 像 spec / 设计 / 看板 / 报告 / 计划 → 倾向 **keep**（加 `--keep`）。
   - 像一次性调试可视化 / 临时 diff / scratch → 倾向 **throwaway**（`--scratch` 或省略）。
   - 说出你的建议，让用户确认或推翻。
4. 调脚本：
   ```bash
   bun "$ROOT/collect-html.ts" add "<abs-path.html>" \
     --project "<name>" --title "<title>" --keep   # 或 --scratch
   ```
5. 把脚本返回的面板 URL 打印给用户。

## Commands

| 意图 | 命令 |
| --- | --- |
| 收藏一个产物 | `add <path> --project <p> --title <t> [--keep\|--scratch]` |
| 清掉过期 scratch | `prune` |
| 移除一条（只断链） | `remove <id>` |
| 列出全部 | `list` |

确定性的活全归脚本：建 symlink、原子更新 manifest、重生成 `index.html`、按日期
prune、按 id 移除。**绝不手改 manifest —— 一律调脚本。**

## Boundaries

- **删除 = 只断链。** 本 skill 和面板都绝不 `rm` 项目真文件。要删源文件，用户去项目里删。
- **面板的 archive / delete** 由 web 面板后端（`scripts/server.ts`）负责，不归本 skill。
  本 skill 只管「收录 + prune」这一侧。

## Manifest contract

`~/collect-html/manifest.json` 是本 skill 与 web 面板共享的契约。每条：
`{ id, title, project, target, disposition, addedAt }`，
其中 `disposition ∈ { kept, scratch, archived }`。

## 默认存储与配置

- 默认 root：`~/collect-html/`（环境变量 `COLLECT_HTML_HOME` 可覆盖）。
- scratch 过期天数：默认 14（`COLLECT_HTML_TTL` 可覆盖）。
- `add` 后无需手动开服务器即可用：脚本生成的 `~/collect-html/index.html` 可
  直接 `file://` 打开浏览。要在面板上点击归档/删除，需启动后端 —— 见 [SETUP.md](SETUP.md)。

## 首次安装面板（可选，一次性）

想要常驻面板 + OrbStack 域名 + 可点击的归档/删除，见 [SETUP.md](SETUP.md)
（原生 `bun server.ts` 最简，或 Docker + OrbStack）。
