# Workflow 1：Brainstorm & Plan（用 HTML 做头脑风暴和实施计划）

## 适用场景

- 用户提了一个题目/想法，希望"想想怎么做"。
- 用户要写 PRD、技术规格、实施方案。
- 用户已经有一份冗长的 Markdown 计划，但自己也不愿意再读了。

## 核心思想

把"长得让人不想读的 Markdown 计划"换成"想点进去看的 HTML 网页"。
人愿意看 → 人会评审 → 协作质量上升 → 最终产品质量上升。

---

## 两步走

### 步骤 1：HTML 头脑风暴页

**目标**：让 Claude 不要再吐文字列表，而是产出一个**可滚动的可视化网页**，里面 5–8 个想法。每个想法包含：

- 标题
- 一段 1–3 句的说明（中文）
- 一个小 mockup（用 div + 简单 CSS 表达视觉概念，**不要插图片**）
- 风险评估卡（高/中/低 + 一句风险描述）
- 一个"为什么有趣"或"潜在收益"小注解

**默认输出**：单文件 HTML，保存到 `./outputs/brainstorm-<slug>.html`。

**给 Claude 的 prompt 模板**（中文版本在 `prompts/brainstorm.md`）：

> 我现在在思考 [题目/上下文]。请你把 5–8 个想法**直接写成一份 HTML 文件**——
> 不要用文字列表。每个想法包含：
> - 标题（中文）
> - 1–3 句说明（中文）
> - 一个用 div/CSS 表达的可视化小 mockup
> - 风险评估（高/中/低 + 一句话）
> - "为什么值得做"的小注解
>
> 整页用中文。简洁现代风格。我相信你的判断，按你认为最有吸引力的方式排版。
> 输出到 `./outputs/brainstorm-<slug>.html`。

---

### 步骤 2：从想法到完整实施计划

**触发**：用户在头脑风暴页里选了一个想法（或者直接告诉你选哪个）。

**做法**：
1. 先简短反问 1–3 个问题，让 Claude 把上下文补全（"用户量级"、"性能要求"、"团队规模"等）。**这一步不要省**——它直接决定了计划的质量。
2. 然后输出一份**完整的实施计划 HTML**。

**这份 HTML 应当包含**：

- **标题与一句话定位**
- **目标与非目标**（哪些做、哪些刻意不做）
- **用户故事**（2–4 条，每条带角色、动机、验收）
- **架构图**：用 div/CSS/SVG 画一个简化的系统架构
- **文件结构**：用 `<pre>` + 缩进展示
- **关键代码片段**：用 `<pre><code>` 高亮（手动加 class，不必引外部库）
- **UI Mockup**：核心页面/组件的 HTML+CSS 草图
- **视觉氛围板（mood board）**：色板 + 字体方向 + 动效原则
- **里程碑与排期**：表格或时间线
- **风险与应对**

**Prompt 模板**：

> 基于头脑风暴页里的"[选定的想法]"，生成完整实施计划。请输出一份单文件 HTML，
> 包含：目标与非目标、用户故事、架构图、文件结构、关键代码片段、UI mockup、
> 视觉 mood board、里程碑、风险与应对。
>
> 给我**最大化的 context** —— excerpts、mockups、code，whatever is needed.
> 整页中文。配色克制。可读性优先。Hey Claude, I trust you here.
> 输出到 `./outputs/plan-<slug>.html`。

## 实战检查清单

写完 HTML 之后，执行这套自检：

- [ ] 标题、正文、按钮文案、注释**全是中文**？
- [ ] mockup 是用 div/CSS 画的，**不是占位灰块**？
- [ ] 至少有一个**图（架构/时间线/数据流）**，不只是文字？
- [ ] 文件能用浏览器直接打开（**没有外部 CDN 依赖**或者依赖明确声明）？
- [ ] 没有 emoji 装饰？没有过度渐变？
- [ ] 一个普通同事打开它，**会不会愿意往下滚**？

如果第 6 条答案不是"会"，重新设计 hero 区。

---

## 反模式（不要做）

- ❌ 把 Markdown 大纲简单包一层 `<div>`。HTML 的价值是**可视化**，不是"换皮"。
- ❌ 引一堆 CDN（Tailwind、AlpineJS 等）。除非用户要求，**保持单文件、零依赖**。
- ❌ 把整页做成深色背景 + 霓虹色。**白底黑字最易读**，不要为了"显得专业"牺牲可读性。
- ❌ 在用户还没选定方向时就生成完整实施计划——**先有头脑风暴，再有计划**。

---

## 何时跳到 Workflow 2

如果用户在浏览实施计划时说："这一段决策规则不对" / "这个表格我想改"，**不要直接在对话里改**——切到 Workflow 2，给他生成一个微应用。
