# Workflow 2：Disposable Micro-App（一次性微应用）

## 适用场景

HTML 计划/方案里出现了**用文本对话改起来很别扭**的内容，例如：

- 一组决策规则（条件 → 动作）
- 一份配置表（多列、多行，互相耦合）
- 一组组件参数（颜色、间距、字号、圆角等）
- 一份分类映射表（输入类型 → 视觉表现）

## 核心思想

> **"This is micro-software on top of micro-software."**
> 既然 token 便宜了，那就为这一次编辑造一个临时定制工具。改完即扔。

让 Claude 不要直接修改原文件，而是先生成一个**专门用来编辑这一段**的 HTML 微应用，让用户在浏览器里点一点、调一调，最后通过"复制配置"按钮把结构化结果输出出来，再回灌到主文件。

---

## 微应用应当包含什么

最低标准：

- **输入控件**：文本框、下拉、滑杆、颜色选择器——按数据类型选择最贴合的。
- **新增 / 删除 / 排序按钮**：动态行/项必备。
- **预览区**：实时显示当前编辑结果（哪怕是简单的 `<pre>` 输出 JSON 也好）。
- **复制按钮**：一键把最终配置（JSON / Markdown / 表格）复制到剪贴板。
- **重置按钮**：一键回到初始状态。

加分项：

- 校验：必填项、数值范围、唯一性。
- 撤销 / 重做。
- 拖拽排序。
- 导入/导出 JSON。

## 给 Claude 的 Prompt 模板（中文）

> 我有这么一段需要编辑的内容：
> ```
> [粘贴具体表格/规则/配置]
> ```
>
> 我想要一个**可编辑的 HTML 微应用**来调整它。我不喜欢现在这个版本。
> 请你：
> - 帮我设计**最适合这个问题**的交互界面（不要默认给我一个表格）。
> - 给我足够的结构（保证不会输入错），但同时给我灵活度。
> - 加一个"复制最终配置"按钮，输出 [JSON / Markdown / 你认为最合适的格式]，
>   方便我粘回主文件。
> - 整页中文。单文件 HTML，零外部依赖。
>
> Hey Claude, I trust you here. Design the ideal interface for this problem.
> 输出到 `./outputs/editor-<slug>.html`。

中文版本完整 prompt 在 `prompts/micro-app.md`。

## 实战检查清单

写完微应用之后：

- [ ] 用户能在不读说明文字的情况下，**3 秒内看出怎么操作**？
- [ ] 修改某一项后，**结果区立刻更新**（不是按"保存"才更新）？
- [ ] 复制按钮真的把数据复制了，并且**给了视觉反馈**（"已复制"）？
- [ ] 输入校验做了（不能为空、数值在合理范围、不允许重复 key 等）？
- [ ] 整页中文？没有 `<input placeholder="enter value">` 这种洋文残留？

---

## 反模式（不要做）

- ❌ 给一个空表格让用户自己填——**没有结构 = 没有价值**。
- ❌ 把所有字段做成 `<input type="text">`——枚举字段用下拉，颜色用 color picker，数值用 number。
- ❌ 没有"复制"按钮——用户调完了不知道怎么把数据带走，工具就废了。
- ❌ 改一个字段引发整页 re-render 闪一下——尽量用细粒度更新。
- ❌ 在微应用里塞品牌 logo、tagline、navbar——**这是一次性工具，不是产品页**。

## 工作流闭环

```
HTML 计划（Workflow 1 产物）
    │
    │  发现某段难改
    ▼
HTML 微应用（Workflow 2 产物）── 用户在浏览器里调整
    │
    │  点"复制最终配置"
    ▼
带回到 HTML 计划，覆盖原段落
    │
    │  扔掉微应用
    ▼
继续 Workflow 1 / 实施
```

完成后**主动建议用户删掉微应用文件**或归档到 `./outputs/_archive/`。微应用的存在意义只在它被使用的那 5 分钟里。
