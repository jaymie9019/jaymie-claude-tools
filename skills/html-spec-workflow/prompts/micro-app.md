# Prompt 模板：一次性微应用（编辑 UI）

> 用于把 HTML 计划里"难以直接手改"的部分变成定制编辑器。

---

## 标准版

```
我有这么一段需要编辑的内容：

```
[粘贴具体表格 / 决策规则 / 配置 / 映射关系]
```

我想要一个**可编辑的 HTML 微应用**来调整它。我不喜欢现在这个版本。

请你：
- 帮我设计**最适合这个问题**的交互界面（不要默认给我一个表格——
  按数据特性选择最贴合的控件）。
- 给我足够的结构（保证不会输入错），同时给我灵活度。
- 加一个"复制最终配置"按钮，输出 [JSON / Markdown / 你认为最合适的格式]，
  方便我粘回主文件。
- 必要的输入校验（必填、范围、唯一性）。
- 整页中文。单文件 HTML，零外部依赖。

Hey Claude, I trust you here. Design the ideal interface for this problem.

输出到 ./outputs/editor-[简短英文 slug].html
```

---

## 简化版

```
帮我做一个一次性 HTML 编辑器，编辑下面这段 [规则 / 配置]：

[粘贴]

要点：贴合数据形态的控件，实时预览，一键复制结果，整页中文。
存到 ./outputs/editor-[slug].html。Hey Claude, I trust you here.
```

## 进阶版（带导入导出）

```
基于下面这段数据：

[粘贴]

做一个 HTML 微应用：
- 默认载入这份数据
- 每个字段用最合适的控件
- 实时预览结果
- 支持「导入 JSON」「导出 JSON」两个按钮
- 支持撤销 / 重做
- 拖拽排序（如果是列表）
- 校验错误时按钮变灰，鼠标悬停显示原因

整页中文。单文件，零依赖。存到 ./outputs/editor-[slug].html。
```
